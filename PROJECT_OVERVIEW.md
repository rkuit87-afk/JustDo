# PROJECT_OVERVIEW.md

# JustDo

## Mission

JustDo exists to create objective accountability while reducing unnecessary administrative burden.

The software should make work easier to perform, easier to justify, and easier to improve through evidence rather than opinion.

The purpose is not surveillance.

The purpose is not bureaucracy.

The purpose is not to create more forms.

The purpose is to replace paperwork with useful tools and replace assumptions with facts.

---

# First Principle

## The Truth comes 

This is the defining principle of the project.

It is not negotiable.

Every feature, workflow, automation and design decision must be evaluated against this principle.

If a decision increases unnecessary burden on the artisan, it should be challenged.

If information can be captured automatically instead of manually, it should be.

If data can be reused instead of entered twice, it should be.

If software creates work instead of removing work, the design has failed.

---

# Accountability

The objective is not to protect artisans.

The objective is not to protect management.

The objective is to protect the truth.

Responsibility should naturally fall where responsibility belongs.

When objective evidence exists:

* Good work should be visible.
* Poor work should be visible.
* Missing planning should be visible.
* Missing procurement should be visible.
* Missing stores support should be visible.
* Missing management should be visible.
* Missing execution should be visible.

The software should reduce politics and increase evidence.

---

# Product Philosophy

Every interaction should answer one question:

"Does this make somebody's job genuinely easier?"

Technology should remove friction.

Technology should eliminate repetition.

Technology should reduce paperwork.

Technology should improve visibility.

Technology should support better decisions through better information.

---

# Data Philosophy

Capture data once.

Reuse it everywhere.

Automate wherever practical.

Avoid duplicate entry.

Avoid unnecessary typing.

Avoid unnecessary clicks.

Never ask a human to provide information that can be obtained automatically.

---

# Design Philosophy

Simple beats clever.

Clear beats impressive.

Reliable beats complicated.

Understandable beats fashionable.

Maintainability beats novelty.

Incremental improvement beats unnecessary rewrites.

---

# Engineering Philosophy

Documentation is part of the product.

Every important decision should have a reason.

Every change should be traceable.

Every feature should have a measurable purpose.

Every system should remain understandable by the next engineer.

---

# Long-Term Vision

Build software that people trust because it reflects reality.

Build software that reduces unnecessary work instead of creating it.

Build software that gives every level of an organization better information without sacrificing the people doing the actual work.

Success is not measured by feature count.

Success is measured by time saved, frustration removed, accountability improved, and trust earned.
